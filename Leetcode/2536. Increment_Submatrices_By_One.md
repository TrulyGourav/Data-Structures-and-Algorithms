# 2536. Increment Submatrices by One — All Approaches + Intuition

## Problem Statement
You are given an n × n matrix initially filled with 0 and multiple queries of the form `[row1, col1, row2, col2]`.  
Each query adds **+1** to all cells inside the corresponding submatrix.

Return the matrix after processing all queries.

---

## Intuition
Brute force updates each cell inside the rectangle → too slow (10k queries × 500×500 matrix).

### Key Insight  
Use the **2D Difference Array Technique**:

- Instead of incrementing every cell,
- Mark **only 4 corners** of the rectangle.
- After all queries, run a **2D prefix sum** → the increments automatically "spread" to the full submatrix.

This reduces each query to **O(1)**.

---

## Conceptual Explanation (2D Diff Array)
For query `(r1, c1) → (r2, c2)`:

```
diff[r1][c1]       += 1
diff[r2+1][c1]     -= 1
diff[r1][c2+1]     -= 1
diff[r2+1][c2+1]   += 1
```

Apply 2D prefix sums to build the final matrix.

---

## Time & Space Complexity
| Step | Complexity |
|------|------------|
| Applying Q diff updates | O(Q) |
| Building prefix sum | O(n²) |
| Total | **O(Q + n²)** |
| Space | O(n²) |

---

## Code (Final Solution)

```java
class Solution {
    public int[][] rangeAddQueries(int n, int[][] queries) {
        int[][] matrix = new int[n][n];

        for (int[] query : queries) {
            int row1 = query[0];
            int col1 = query[1];
            int row2 = query[2];
            int col2 = query[3];

            matrix[row1][col1]++;

            if (row2 + 1 < n) matrix[row2 + 1][col1]--;
            if (col2 + 1 < n) matrix[row1][col2 + 1]--;
            if (row2 + 1 < n && col2 + 1 < n) matrix[row2 + 1][col2 + 1]++;
        }

        for (int row = 0; row < n; row++) {
            for (int col = 0; col < n; col++) {
                if (row > 0) matrix[row][col] += matrix[row - 1][col];
                if (col > 0) matrix[row][col] += matrix[row][col - 1];
                if (row > 0 && col > 0) matrix[row][col] -= matrix[row - 1][col - 1];
            }
        }

        return matrix;
    }
}
```

## All Approaches Summary

| Approach | Idea | Time | Status |
|----------|------|------|--------|
| Brute Force | Update every cell of submatrix | O(Q × n²) | ❌ |
| Row-wise Update | Update only rows | O(Q × n) | ❌ |
| **2D Difference Array** | Mark corners + prefix sum | **O(Q + n²)** | ✅ Best |
| Fenwick Tree | Range updates | O(log²n) | Overkill |
| Segment Tree | Lazy rectangle | O(log²n) | Overkill |

## Memory Tip (Never Forget)
**Think of each query as a water hose starting at (r1, c1).**  
You turn it ON at the start and OFF at boundaries.  
2D prefix sum makes the water naturally fill the entire rectangle.

👉 **“Mark the corners → Let prefix fill the rectangle.”**

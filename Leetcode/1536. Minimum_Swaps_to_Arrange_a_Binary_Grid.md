# 🚀 Shared Local Infrastructure (Dev Only)

This folder contains a **shared Docker-based local infrastructure** used
by multiple microservices.

It provides:

-   🐘 PostgreSQL
-   🧠 Redis
-   📡 Kafka
-   🐘 Zookeeper

⚠️ This infrastructure is generic and does NOT create any
project-specific databases automatically.

Each microservice must create and manage its own database.

------------------------------------------------------------------------

# 📁 Folder Structure

    dev-infra/
     ├── docker-compose.yml
     ├── .env
     └── README.md

------------------------------------------------------------------------

# 🧠 Architecture Overview

Single infrastructure → Multiple databases → Multiple microservices

Example:

-   wallet-service → walletledger DB
-   payment-service → payment_db
-   notification-service → notify_db

------------------------------------------------------------------------

# 🛠️ First-Time Setup

## 1️⃣ Navigate to infra folder

``` bash
cd dev-infra
```

------------------------------------------------------------------------

## 2️⃣ Configure environment variables

Open `.env` and verify:

    POSTGRES_USER=devuser
    POSTGRES_PASSWORD=devpass
    POSTGRES_PORT=5432

    REDIS_PORT=6379
    ZOOKEEPER_PORT=2181
    KAFKA_EXTERNAL_PORT=9092

------------------------------------------------------------------------

## 3️⃣ Start Infrastructure

``` bash
docker compose up -d
```

Verify running containers:

``` bash
docker ps
```

You should see:

-   shared-postgres
-   shared-redis
-   shared-kafka
-   shared-zookeeper

------------------------------------------------------------------------

# 🐘 Create Database for a Microservice

⚠️ This must be done once per project.

Example: wallet-service

### Step 1 --- Enter Postgres container

``` bash
docker exec -it shared-postgres psql -U devuser
```

### Step 2 --- Create Database

Inside psql:

``` sql
CREATE DATABASE walletledger;
```

Exit:

``` sql
\q
```

------------------------------------------------------------------------

# 🔌 Microservice Configuration

## If running service locally (not inside Docker)

Example Spring Boot:

    spring.datasource.url=jdbc:postgresql://localhost:5432/walletledger
    spring.datasource.username=devuser
    spring.datasource.password=devpass

------------------------------------------------------------------------

## If running service inside Docker

Use container hostname:

    spring.datasource.url=jdbc:postgresql://shared-postgres:5432/walletledger

------------------------------------------------------------------------

# 📡 Kafka Configuration

Local:

    localhost:9092

Inside Docker:

    shared-kafka:29092

------------------------------------------------------------------------

# 🧠 Redis Configuration

Local:

    localhost:6379

Inside Docker:

    shared-redis:6379

------------------------------------------------------------------------

# 🛑 Stop Infrastructure

    docker compose down

This stops containers but keeps data.

------------------------------------------------------------------------

# 🗑 Reset Everything (WARNING: Deletes All Data)

    docker compose down -v

This deletes:

-   Postgres data
-   All databases
-   All stored messages

------------------------------------------------------------------------

# 🔍 Check Existing Volumes

    docker volume ls

Remove unused volumes:

    docker volume prune

------------------------------------------------------------------------

# 🧱 Recommended Workflow for New Microservice

1.  Start shared infra\
2.  Create new database\
3.  Configure service DB connection\
4.  Run migrations\
5.  Start service

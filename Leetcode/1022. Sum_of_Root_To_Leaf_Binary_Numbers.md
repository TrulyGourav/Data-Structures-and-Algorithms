# 1022. Sum of Root To Leaf Binary Numbers

------------------------------------------------------------------------

## 🧠 Problem Statement

You are given the root of a binary tree where each node has a value `0`
or `1`.\
Each root-to-leaf path represents a binary number starting with the
**most significant bit (MSB)**.

For example:

Path: `0 → 1 → 1 → 0 → 1`\
Binary number formed: `01101`\
Decimal value: `13`

Return the **sum of all root-to-leaf binary numbers**.

Constraints: - Each node value is `0` or `1` - Answer fits in 32-bit
integer

------------------------------------------------------------------------

# 🌟✨ INTUITION (MOST IMPORTANT PART)

👉 Every time we move from parent → child, we are adding a new bit at
the end of our current binary number.

In binary: - Adding a new digit to the right means:

        previous_number * 2 + current_bit

Why?

Because shifting left in binary = multiply by 2.

So instead of storing path as string and converting later ❌\
We directly build the decimal value while traversing ✅

------------------------------------------------------------------------

# 🔍 Conceptual Understanding

This is a **Tree + DFS + Bit Manipulation** problem.

At each node:

    current_value = previous_value * 2 + node.val

When we reach a leaf: - Add that number into global sum.

We don't need extra space for path storage.

------------------------------------------------------------------------

# 🌳 Recursion Tree Visualization

Example Tree:

            1
           /       0   1
         / \       0   1   1

Paths:

1 → 0 → 0 = 100 (4)\
1 → 0 → 1 = 101 (5)\
1 → 1 → 1 = 111 (7)

Total = 4 + 5 + 7 = 16

------------------------------------------------------------------------

### Recursion Flow (for path 1 → 0 → 1)

    dfs(1, 0)
     val = 0*2 + 1 = 1

        dfs(0, 1)
         val = 1*2 + 0 = 2

            dfs(1, 2)
             val = 2*2 + 1 = 5
             Leaf → add 5

------------------------------------------------------------------------

# ⏱ Time & Space Complexity

### Time Complexity:

O(N)\
Each node visited exactly once.

### Space Complexity:

O(H)\
H = height of tree (recursion stack)

Worst case (skewed tree): O(N)\
Balanced tree: O(log N)

------------------------------------------------------------------------

# 💻 Code Solution (DFS)

``` java
class Solution {
  public int sumRootToLeaf(TreeNode root) {
    dfs(root, 0);
    return ans;
  }

  private int ans = 0;

  private void dfs(TreeNode root, int val) {
    if (root == null)
      return;
      
    val = val * 2 + root.val;

    if (root.left == null && root.right == null)
      ans += val;

    dfs(root.left, val);
    dfs(root.right, val);
  }
}
```

# 🧵 Other Possible Approaches

1.  Brute Force (Store Path as String)
    -   Build string for each path
    -   Convert binary string to decimal at leaf
    -   Time: O(N²) worst case (string operations)
    -   Space: O(N)
2.  Store Path in List and Compute at Leaf
    -   Maintain list of bits
    -   Compute number at leaf
    -   Time: O(N²)
    -   Space: O(N)
3.  Iterative DFS using Stack
    -   Same logic but no recursion
    -   Time: O(N)
    -   Space: O(H)
4.  BFS Level Order with Pair\<Node, Value\>
    -   Queue based solution
    -   Time: O(N)
    -   Space: O(N)
5.  🔥 Current Optimal Solution (DFS + Bit Accumulation) ← (OPTIMAL)
    -   Time: O(N)
    -   Space: O(H)

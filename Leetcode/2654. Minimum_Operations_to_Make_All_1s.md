# 🧩 Minimum Number of Operations to Make All Array Elements Equal to 1

**LeetCode 2654 | Medium | Array, Math (GCD)**  
🔗 [LeetCode Link](https://leetcode.com/problems/minimum-number-of-operations-to-make-all-array-elements-equal-to-1/)

---

## 🧠 Problem Statement

You are given a **0-indexed** array `nums` consisting of positive integers.  
You can perform the following operation **any number of times**:

> Select an index `i` such that `0 <= i < n - 1`, and replace **either** `nums[i]` **or** `nums[i + 1]` with `gcd(nums[i], nums[i + 1])`.

Return the **minimum number of operations** required to make **all elements equal to 1**.  
If it is **impossible**, return `-1`.

---

### 📘 Example 1
```
Input:  nums = [2,6,3,4]
Output: 4
```
**Explanation:**
1. gcd(3,4) → replace nums[2] with 1 → [2,6,1,4]  
2. gcd(6,1) → replace nums[1] with 1 → [2,1,1,4]  
3. gcd(2,1) → replace nums[0] with 1 → [1,1,1,4]  
4. gcd(1,4) → replace nums[3] with 1 → [1,1,1,1]

---

### 📘 Example 2
```
Input:  nums = [2,10,6,14]
Output: -1
```
**Explanation:**  
No matter how we apply operations, GCD will never reach 1. Hence, impossible.

---

## 🧩 Intuition

Think of the **GCD** as a "compressor" —  
it reduces two numbers but **never increases** them.  
The **only way to reach `1`** is if there exists at least **one contiguous subarray** with GCD = 1.  
Once you can make a single `1`, you can **spread that 1** across the array by applying more operations.

So, our task becomes:
1. **Check if we already have any 1s** → If yes, just make others 1.
2. If not, **find the shortest subarray whose GCD = 1**, because that’s the minimal effort to first produce a `1`.

After that, converting all other elements to 1 takes `(n - 1)` more steps.

---

## 🧮 Conceptual Understanding

Let’s visualize the operation using a small test case:  
`nums = [2,6,3,4]`

```
Initial: [2,6,3,4]
          |   | |
GCDs:   gcd(2,6)=2
         gcd(6,3)=3
         gcd(3,4)=1 ✅  <-- Found subarray giving GCD = 1 (from index 2-3)
```

This tells us:  
- We can first make one element 1 using 2 operations (distance between indices).  
- Then, spread it left and right to make all 1s.

---

### 🌳 GCD Exploration Tree (for intuition)

Example: nums = [6, 10, 15]

```
(6,10,15)
   |
   | gcd(6,10)=2
   | gcd(10,15)=5
   ↓
 [2,10,15] or [6,5,15]
     ↓            ↓
  gcd(2,10)=2   gcd(5,15)=5
       ↓             ↓
     never reach 1 → impossible
```
Hence, no subarray gives GCD = 1 → return -1.

## 💻 Code Implementation

```java
class Solution {
  public int minOperations(int[] nums) {
    final int n = nums.length;
    final int ones = (int) Arrays.stream(nums).filter(num -> num == 1).count();
    if (ones > 0)
      return n - ones; // Spread existing 1s to make all 1

    int minOps = Integer.MAX_VALUE;

    // Find the shortest subarray whose GCD == 1
    for (int i = 0; i < n; ++i) {
      int g = nums[i];
      for (int j = i + 1; j < n; ++j) {
        g = gcd(g, nums[j]);
        if (g == 1) {
          minOps = Math.min(minOps, j - i);
          break;
        }
      }
    }

    // If never found, impossible
    return minOps == Integer.MAX_VALUE ? -1 : minOps + n - 1;
  }

  private int gcd(int a, int b) {
    return b == 0 ? a : gcd(b, a % b);
  }
}
```

## ⏱️ Time & Space Complexity

| Operation | Complexity | Explanation |
|------------|-------------|-------------|
| Finding 1s | O(n) | Count number of 1s |
| GCD Subarray Search | O(n² * logM) | For each pair, we may compute gcd (logM = cost of gcd) |
| Overall | **O(n² logM)** | n ≤ 50 → easily acceptable |
| Space | **O(1)** | Only constant space used |

## 🧮 Other Possible Approaches

| Approach | Idea | Time | Space | Note |
|-----------|------|------|-------|------|
| Brute Force | Try all possible operations recursively | Exponential | O(n) | Not feasible |
| Optimized GCD Scan (**Current**) | Find shortest subarray with GCD = 1 | **O(n² logM)** | O(1) | ✅ **Used here** |
| Segment Tree (GCD Range Query) | Precompute GCD of segments to speed up search | O(n log n) | O(n) | Overkill for n ≤ 50 |
| Sliding Window GCD | Expand window and keep updating gcd | O(n² logM) | O(1) | Same as current but alternative implementation |

## 🧭 Pattern Recognition
- This problem fits under **“GCD propagation”** or **“Reduce-by-GCD”** patterns.
- Core trick: **Find minimal subarray with GCD == 1** → “seed” the array with a 1.
- Once a single `1` appears, the rest is trivial spreading.

You can recognize similar problems when:
- You’re asked to make all elements same using GCD/LCM operations.
- You see “replace with gcd(a,b)” operations → think *how to reach 1*.

## 🧠 Tip to Remember (🔥 Never Forget)

> **Think of the GCD like a disease spreading across the array.**
> Only when one cell becomes “infected” with `1`, it can spread rapidly.  
> So first **find the shortest infection point (subarray giving GCD=1)**,  
> then **spread the infection** across the array.  
>  
> 💡 If no such infection can start (no subarray with GCD=1) → impossible!

This mental image will help you **recall instantly** in interviews:
> "Find subarray with GCD=1 → Spread it → n-1 more ops."
